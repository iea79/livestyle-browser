## LiveStyle Sync (браузерное расширение)

Расширение для браузера (Chrome/Chromium), которое принимает CSS из редактора (Cursor/VS Code) через WebSocket и **подставляет стили в выбранную вкладку без перезагрузки**. Работает в паре с расширением редактора из папки `livestyle-editor`.

### Установка и запуск в режиме разработки

1. Установите зависимости и соберите распакованную версию:

```bash
cd /media/job/livestyle-browser
yarn build:unpacked
```

Команда создаст папку `dist-unpacked/` с копией файлов расширения.

2. Загрузите расширение в Chrome / Chromium:
   - Откройте `chrome://extensions`
   - Включите **Режим разработчика**
   - Нажмите **Загрузить распакованное**
   - Выберите папку `dist-unpacked`

3. Запустите сервер LiveStyle в редакторе:
   - Установите расширение из папки `livestyle-editor` (см. его `README.md`)
   - В Cursor/VS Code выполните команду **LiveStyle Sync: запустить сервер**

4. Подключите браузер к редактору:
   - Откройте popup расширения (иконка LiveStyle)
   - В настройках (ссылка «Настройки (URL)» в popup) убедитесь, что указан адрес `ws://127.0.0.1:8765` (порт должен совпадать с настройкой расширения редактора)
   - В popup нажмите **«Включить синхронизацию»** на вкладке, которую хотите править

После этого изменения в CSS / SCSS / Less в редакторе будут пересчитываться расширением редактора и отправляться в браузер, где этот браузерный плагин применяет их к выбранной вкладке.

### Сборка `.crx`

Для локальной установки `.crx` можно собрать подписанный архив:

```bash
cd /media/job/livestyle-browser
yarn build
```

В результате появится `dist/livestyle-sync.crx`. Для сборки используется глобально установленный `crx3` и приватный ключ `.crx-key.pem` (лежит в корне проекта и **не должен попадать в публичный репозиторий**).

### Структура проекта

- `manifest.json` — манифест Chrome‑расширения (MV3)
- `background.js` — service worker, создаёт offscreen‑документ и держит постоянное WebSocket‑соединение с редактором, пересылает CSS во вкладку
- `offscreen.html` / `offscreen.js` — offscreen‑документ, в котором крутится постоянный WebSocket (service worker сам по себе не может жить вечно)
- `content.js` — подставляет полученный CSS в страницу через `<style id="livestyle-sync-injected">`
- `popup.html` / `popup.js` — popup с кнопкой включения/выключения синхронизации для текущей вкладки и ссылкой в настройки
- `options.html` / `options.js` — страница настроек (URL WebSocket редактора, отладочные логи)
- `scripts/build-unpacked.js` — скрипт для сборки распакованной версии `dist-unpacked/` (для загрузки через «Загрузить распакованное»)

### Связка с расширением редактора

Расширение редактора (`/media/job/livestyle-editor`):

- поднимает WebSocket‑сервер (по умолчанию `ws://127.0.0.1:8765`)
- отслеживает изменения в SCSS / Less / CSS
- по необходимости компилирует их в CSS и **патчит указанные скомпилированные CSS‑файлы по селекторам**, после чего отправляет результат в браузер.

Браузерное расширение из этого проекта:

- устанавливает одно постоянное WebSocket‑соединение через offscreen‑документ
- принимает CSS (по одному сообщению на файл) и подставляет стили в выбранную вкладку.

### Что не коммитим

- `dist/` и `dist-unpacked/` — артефакты сборки
- `node_modules/` — зависимости
- `*.crx`, `*.vsix`, временные архивы (`browser.zip` и т.п.)
- приватный ключ `.crx-key.pem` **никогда не должен попадать в публичные репозитории**
